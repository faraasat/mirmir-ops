# Roadmap

## Phase 1: Foundation

- Define the permission tiers and action schema
- Build the extension shell and agent panel
- Implement read-only context extraction
- Add action logging and user-visible audit trail

## Phase 2: MVP Flow

- Enable cross-tab coordination and background tabs
- Add local preference memory and simple retrieval
- Implement micro-confirmations for mutable actions
- Deliver the Tokyo trip workflow end to end

## Phase 3: Trust and Control

- Add fine-grained, per-site permission controls
- Introduce time-bounded consent and revocation
- Improve transparency with step-by-step previews

## Phase 4: Usability and Scale

- Expand to additional workflows beyond travel
- Add configurable intent templates
- Improve performance and reliability on complex sites

## Phase 5: Ecosystem

- Define a developer API for website-level tools
- Allow optional, user-approved integrations
- Publish best practices for safe, cross-site execution
